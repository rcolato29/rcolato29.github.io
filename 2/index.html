Project 2: Fun with Filters and Frequencies

Project Overview

This project explores various image processing techniques using filters and frequency domain operations. The goal is to understand how different kinds of filters affect images and how we can tweak images in both spatial and frequency domains in order to see our desired visual effects.

Some of the main tasks in this project involve implementing 2-dimensional convolutions from scratch, deriving the finite difference operator and derivative of gaussian (DoG) filter. Image blurring, sharpening, hybrid image creation, as well as multi-resolution blending were all implemented. These operations demonstrate fundamental concepts in digital image processing and computer vision.

Convolutions from Scratch

The first part of this project involves implementing convolutions from scratch, using only NumPy operations. There are multiple ways to do this. Below are two approaches: a naive approach uses four for loops and a vectorized approach uses two, where the runtime of the latter is improved significantly. In both cases, the process is as follows: pad the images with zeros, flip the kernel (this distinguishes convolutions from cross-correlation), iterate through the entire image in row-major order, sum up the elementwise products between the image and the kernel within each window, return the convolved image.

Derivative of Gaussian (DoG) Filter

Since having the difference operator alone has some difficulties getting rid of the noise in the results, I used the Gaussian filter to first blur the image, then compute the binarized gradient magnitude. By doing this, one can get rid of the noise in the image and capture the most prominent edges. This can also be done by creating derivate of gaussian (DoG) filters, where the gaussian is convolved with D_x and D_y, then used to compute the gradient magnitude image. The results of both methods are shown below (which achieve very similar results), as well as the filters themselves. For the parameters of the Gaussian filter, I ended up choosing the kernel size to be 25 and the sigma to be 5; A larger kernel size equates to a larger blurring window during convolutions, and a larger sigma value means that the kernel's underlying Gaussian distribution has more spread, which means that the blur is more pronounced. I found that these values gave me enough blur in order to effectively reduce a lot of the noise.

Image "Sharpening"

Now that we can do basic convolutions on images, it is now possible to take a blurry image and "sharpen" it. This is done using the unsharp masking technique. The idea is to construct a high-pass filter in order to extract the highest frequencies of an image, which make the image look sharp. This high-pass filter is derived by subtracting the blurred image from the original image. These high frequencies are added to the original image to get a sharp look. You can adjust the intensity of the sharpness by scaling these high frequencies by a parameter, alpha. In the examples that follow, alpha was chosen to be 5, which strikes a balance between modest sharpening and the distorted "deep-fried" look associated with higher parameter values. In addition, this entire process can be combined into a single convolution operation called the unsharp mask filter, which is the result of the rightmost equality in the derivation below. Hybrid Images

Now that we can derive low-pass and high-pass filters to blur and sharpen images respectively, it is possible to create hybrid images, or images that change based on the viewing distance. This is possible by taking the high frequency portion of one image and stacking it on top of the lower frequency portion of another image. In order to find the optimal cutoff frequencies and achieve the best resulting hybrid images, I chose the following parameters for blurring the first image and sharpening the second: kernel size = 25, alpha = 3, sigma1 = 5, sigma2 = 5. After much experimentation, I found that this specific configuration of parameters made both images noticable at their respective viewing distances. Gaussian Stacks, Laplacian Stacks, and Multiresolution Blending

Now, the goal is to implement multi-resolution blending, or joining two images across a smooth seam, called an image spline. This is done by computing this seam for multiple bands across the entire range of image frequencies. The process is quite involved, and is as follows: create laplacian stacks for both images, then create a gaussian stack for the mask (which will blur the seam and make the transition smooth). Now, on a layer-by-layer basis, add the multiplication of the blurred mask and the lth image of image 1's laplacian stack to the multiplication of the blurred mask's complement and the lth image of image 2's laplacian stack (this step is outlined in the formula below). Finally, sum up all the levels of resulting image blends to generate the final, blended image. This entire process is necessary in order to ensure that a smooth transition between the two images is created.

The Formula for the lth Layer of the Blended Image

The first step is implementing Gaussian stacks, which are a series of images where the Gaussian filter is successively applied, one after the other (without subsampling). Here is an example of these operations on a car:

Next, we need to implement Laplacian stacks, which are comprised of the image differences between two adjacent images in the Gaussian stack. Note that we capture the lowest frequencies by taking the last image in the Gaussian stack.

Finally, we can generate the oraple, which is just a multi-resolution blend between an apple and an orange. The entire process is displayed below: each row represents a layer of the mutliresolution blend, where the left image is the mask applied to image 1, the middle image is the mask complement applied to image 2, and the right image is the addition of the two. As you move down the rows, you also move down in the frequency bands of the image. The bottom-most row shows the resulting blends of that image's respective column, and the bottom right image is the final result.

Now, we have the tools to blend a variety of images together. Further, these two images do not have to be joined across a vertical seam like the oraple. You can also construct a circular seam, which is generated by specifying a radius and zeroing out all pixels in the mask that are further than the radius away from the image center. This is more involved than the vertical seam, which simply zeroes out all pixels in the mask that are on the right-half of the image.
